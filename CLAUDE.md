# 项目开发与测试规范

## 核心要求（CRITICAL）

### 正确理解“浏览器”

如果我在对话中提到了“访问浏览器/打开浏览器”等内容，这表示 **你需要使用 Chrome MCP 对我的浏览器进行操作**（例如获取当前页面快照等）。

### 严禁删除数据

除非我要求你这么做，否则你不能 **删除** 或者 **大规模修改** 单词本内的单词。

### 必须进行测试

每次你完成任何代码编写、功能开发或 Bug 修复后，**严禁直接结束任务**。你必须严格执行以下测试流程：

1. 确认本地开发服务器已经启动（如需启动，请先运行相关命令）。
2. **强制调用 Chrome MCP** 工具打开浏览器。
3. 访问本地测试地址（如 http://localhost:3000）。
4. 在浏览器中实际操作并验证你刚刚修改的功能。
5. 在控制台输出你的测试结果。如果发现报错或 UI 异常，立即进行修复。
6. **如果启动项目时发现 localhost:3000 已经被别的进程占用，说明我已经手动在该项目下运行了 `npm run dev`，请不要重新开一个新的服务，直接用我的服务进行测试就可以了。**
7. **CRITICAL：如果你发现浏览器已经有一个页面是 localhost:3000，请不要再次打开一个新的页面！！！直接对着这个页面进行测试！！！**

**注意：这是不可妥协的硬性规定。未完成 Chrome MCP 测试前，不准告诉我任务已完成。**
