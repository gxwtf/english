目前的“图像单词识别”算法存在如下问题：

- 准确度、精确度、召回率都非常低，即使调节超参数也无济于事，感觉这个问题不是靠调参数能够解决的。

新的图像识别架构如下：

- 用某些专业的、高准确率的 OCR 库，将图片中的单词识别出来，并用 bbox 标记。需要记录图片中每个 bbox 的顶点坐标，以及这个 bbox 的单词。
- 将图片分块，每个块在一个 bbox 的基础上向四周拓展一定距离，以识别某些标记（如红笔圈出、红下划线等）。为了避免错误识别多个单词，需要保证每个块与其它 bbox 的交集大小不能超过该 bbox 的 50%。这一步可以用扫描线等算法实现，注意应该将复杂度控制在 $O(n^2)$ 以下，其中 $n$ 为 bbox 数量。
- 依次识别每个块是否包含用户指定的标签。这一步既可以直接调用 27B 的大模型，也可以先用传统 CV 先排除一些明显不可能的块，再将剩余的块插入 27B 大模型。
