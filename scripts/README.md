# 批量更新单词词性脚本

这个目录包含了用于批量更新单词词性的脚本工具。

## 📋 脚本列表

### 1. `preview-meaning-update.ts` - 预览更新

**用途**: 预览词性更新效果，生成详细报告，不实际修改数据库

**执行**:
```bash
npx tsx scripts/preview-meaning-update.ts
```

**输出**:
- 控制台显示匹配统计和示例
- `meaning-update-report.json` - 详细的 JSON 报告
- `meaning-update-preview.sql` - SQL 预览文件

**适用场景**:
- 首次执行前预览效果
- 检查匹配质量
- 识别需要人工确认的单词

---

### 2. `batch-update-meanings.ts` - 批量更新

**用途**: 批量更新所有单词的词性信息

**执行**:
```bash
npx tsx scripts/batch-update-meanings.ts
```

**特点**:
- 智能匹配用户保存的含义与字典释义
- 批量处理，性能优化
- 详细的进度显示和统计信息

**适用场景**:
- 首次迁移数据
- 全量更新所有单词

---

### 3. `incremental-update-meanings.ts` - 增量更新

**用途**: 只更新缺少词性信息的单词

**执行**:
```bash
npx tsx scripts/incremental-update-meanings.ts
```

**特点**:
- 只处理 `type` 为空的单词
- 保留已有词性信息
- 更快速，更安全

**适用场景**:
- 日常维护
- 补充缺失的词性
- 定期更新

---

## 🚀 推荐执行流程

### 首次迁移

```bash
# 1. 预览更新效果
npx tsx scripts/preview-meaning-update.ts

# 2. 检查报告，确认匹配质量
# 查看 meaning-update-report.json

# 3. 执行批量更新
npx tsx scripts/batch-update-meanings.ts

# 4. 验证更新结果
# 在应用中检查单词列表
```

### 日常维护

```bash
# 定期执行增量更新
npx tsx scripts/incremental-update-meanings.ts
```

## ⚙️ 配置选项

编辑 `batch-update-config.ts` 可以调整：

- `batchSize`: 批量处理大小（默认 50）
- `delayBetweenQueries`: 查询延迟（默认 10ms）
- `matchThreshold`: 匹配阈值（默认 0.6）
- `updateStrategy`: 更新策略（preview-only/all/missing-only）

---

## 🔍 匹配算法

脚本使用多级匹配策略：

1. **完全匹配**: 文本完全相同
2. **包含匹配**: 一个文本包含另一个
3. **相似度匹配**: Jaccard 相似度 > 0.8

匹配成功后，保留用户原始的含义文本，只补充词性信息。

---

## ⚠️ 注意事项

1. **数据备份**: 执行前建议备份数据库
2. **预览优先**: 首次执行务必先运行预览脚本
3. **检查报告**: 仔细检查低匹配度的单词
4. **性能考虑**: 大量单词可能需要较长时间
5. **字典依赖**: 需要字典文件 `src/dict/ecdict.csv`

## 🛠️ 故障排查

### 问题: 字典查询失败

**检查**:
- 字典文件是否存在: `src/dict/ecdict.csv`
- 文件路径是否正确

### 问题: 匹配度低

**解决**:
- 检查 `meaning-update-report.json` 中的低匹配单词
- 手动确认这些单词的含义
- 考虑调整 `matchThreshold` 参数

### 问题: 更新速度慢

**优化**:
- 减小 `delayBetweenQueries`
- 增大 `batchSize`
- 使用增量更新代替全量更新

## 📝 开发说明

编辑 `matchMeanings()` 函数，添加自定义匹配逻辑。

### 自定义报告格式

修改脚本中的报告生成部分，添加需要的字段。

### 集成到 CI/CD

可以将增量更新脚本集成到定时任务中，定期补充词性信息。
